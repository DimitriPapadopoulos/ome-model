Introduction

This document outlines the XML Schemas created by the Open Microscopy Environment Group. Detailed information on the schemas and future development plans are available at http://www.ome-xml.org/. Further information on the groups work is available at http://www.openmicroscopy.org/

History

This document describes the current XML Schemas using the current version scheme. For information on the schemas released before June 2007 is available at http://www.openmicroscopy.org/XMLschemas/. This includes the schemas used by the origional OME server.

Schemas

This has the schema name, the current version, a link to the XSD file, and a short description

  • AnalysisChain - June 2007 Version 1 [AnalysisChain.xsd]The Analysis Chain schema. Analysis chains are how module outputs are connected to inputs of other modules.
  • AnalysisModule - June 2007 Version 1 [AnalysisModule.xsd] The Analysis Module Library schema. Analysis modules are image analysis algorithms or, more generally, how information is generated in OME. Makes use of CLI and MLI schemas.
  • BinaryFile - June 2007 Version 1 [BinaryFile.xsd] The Binary File schema is used to describes a file location, or the location of a fragment within a file.
  • CA - June 2007 Version 1 [CA.xsd] The standard custom attribute schema.
  • CLI - June 2007 Version 1 [CLI.xsd]The Command Line Interface sub-schema.
  • DataHistory - June 2007 Version 1 [DataHistory.xsd]The Data History schema. Data history describes the data dependency in OME.
  • MLI - June 2007 Version 1 [MLI.xsd]The Matlab Interface sub-schema.
  • OME - June 2007 Version 1 [ome.xsd] The main schema which defines the OME ontology for microscopy. This schema makes use of all of the others.
  • SPW - June 2007 Version 1 [SPW.xsd] Short Description
  • STD - June 2007 Version 1 [STD.xsd]The Semantic Type definition schema. Once a Semantic Type is defined, data for the defined type can be imported into the OME server.
