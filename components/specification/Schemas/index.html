Introduction

This document outlines the XML Schemas created by the Open Microscopy Environment Group. Detailed information on the schemas and future development plans are available at http://www.ome-xml.org/. Further information on the groups work is available at http://www.openmicroscopy.org/

Auto generated documentation for the schemas is is available on the development server: http://cvs.openmicroscopy.org.uk/svn/specification/Documentation/Generated/OME-2008-09/ome.xsd.html and http://cvs.openmicroscopy.org.uk/svn/specification/Documentation/Generated/OME-2008-09/OMERO.xsd.html

History

This document describes the current XML Schemas using the current version scheme.

  • June 2007 - Major release - new namespace /2007-06/
  • September 2007 - Minor release - version number increment
  • February 2008 - Major release - new namespace /2008-02/
  • May 2008 - Minor release - version number increment
  • September 2008 - Major release - new namespace /2008-09/

For information on the schemas released before June 2007 is available at http://www.openmicroscopy.org/XMLschemas/. This includes the schemas used by the original OME server.

Schemas

This has the schema name, the current version, a link to the XSD file, and a short description

  • AnalysisChain - September 2008 Version 1 [AnalysisChain.xsd] The Analysis Chain schema. Analysis chains are how module outputs are connected to inputs of other modules.
  • AnalysisModule - September 2008 Version 1 [AnalysisModule.xsd] The Analysis Module Library schema. Analysis modules are image analysis algorithms or, more generally, how information is generated in OME. Makes use of CLI and MLI schemas.
  • BinaryFile - September 2008 Version 1 [BinaryFile.xsd] The Binary File schema is used to describes a file location, or the location of a fragment within a file.
  • CA - September 2008 Version 1 [CA.xsd] The standard custom attribute schema.
  • CLI - September 2008 Version 1 [CLI.xsd] The Command Line Interface sub-schema.
  • DataHistory - September 2008 Version 1 [DataHistory.xsd] The Data History schema. Data history describes the data dependency in OME.
  • MLI - September 2008 Version 1 [MLI.xsd] The Matlab Interface sub-schema.
  • OME - September 2008 Version 1 [ome.xsd] The main schema which defines the OME ontology for microscopy. This schema makes use of all of the others.
  • OMERO - September 2008 Version 1 [OMERO.xsd] The schema extensions used by OMERO. It is not used by the OME schema but provides a structure for data in StructuredAnnotation blocks used by OMERO.
  • SA - September 2008 Version 1 [SA.xsd] The Structured Annotation schema.
  • SPW - September 2008 Version 1 [SPW.xsd] The Screen/Plate/Well schema.
  • STD - September 2008 Version 1 [STD.xsd] The Semantic Type definition schema. Once a Semantic Type is defined, data for the defined type can be imported into the OME server.
